#### 添加注释
```
请为下面的 Python 代码修正代码风格并写或重写注释。对于修正代码风格，你能修改的仅限于排版和变量名，不允许修改代码实现、逻辑等其他全部内容。对于写或重写注释，严格模仿 NumPy docstring（numpydoc）风格，并保持与实现一致。只输出结果，不要解释，不要额外文字。
注释要求：
1) 使用英语注释。
2) 函数、类、类的__init__、以及类的方法都要有三引号 docstring，放在定义后的第一行。
3) 类 docstring：只用 1 句话概述用途（不要展开细节）。
4) 函数、类的__init__和类的方法 docstring 的结构固定：
   - 1–3 句说明做什么；必要时说明张量形状/坐标含义或关键数学定义。
   - 然后写：
     Parameters
     ----------
     <param_name> : <type>
         <说明：语义、shape、约束、默认值或取值范围（如适用）>
     ...

     Returns
     -------
     <return_name> : <type>
         <说明：返回对象的语义与形状>
     ...
4) 参数与返回值使用 “name : type” 格式；说明行缩进 4 个空格。
5) 在函数体内部只写少量行内注释（#），用于标记关键阶段或意图，每条尽量短，不写长段落。
6) 类中的类初始化调用使用（##）写少量注释，写在定义的前一行
7) 术语保持一致，不要同义词反复替换。
8) 注释必须与代码逻辑完全一致；如果代码与旧注释冲突，以代码为准修正注释。

代码风格要求：  
1）在每个类定义之前保留两行空行；在每个函数定义之前保留一行空行
2）import 部分先导入库模块，再导入项目中实现的类、函数或其他对象，这两部分之间保留一行空行。
3）仅在不满足如下标准时修改命名：  
   - 类名统一使用驼峰法（CamelCase），首字母大写。  
   - 类的实例名统一使用 snake_case（下划线命名法）。  
   - 普通变量名与函数名统一使用 snake_case（下划线命名法）。  
4）所有函数的参数列表（def 的括号内）必须逐参数换行，并按“变量名: 变量类型”的形式标注类型；如果该参数原本有默认值“= ...”，必须保留默认值。对于可空类型，不要使用 type | None，统一改为 Optional[type] = None
5）在调用函数时，默认直接传入参数值，不要写成带参数名的形式。但在以下场景中，应显式写出参数名，使用 参数名=参数值 的形式：
   - 类的实例化或构造器调用。
   - 参数较多，使用关键字参数能显著提升可读性。
   - 配置项传递、超参数设置或语义容易混淆的调用。
   - 需要避免位置错误或增强代码可维护性时。
6）当函数调用、容器初始化或推导式表达式较长时，采用括号内分层换行的写法。
7）类中的辅助函数（仅供类内部使用的 helper/private 方法）统一改为 _function_name 格式（在函数名前加单下划线）；对外接口方法不要加下划线。  
8）以下具体变量命名规则必须统一执行：  
- 对于 numpy 或 torch 的矩阵/张量变量，“维度数量”对应的变量名必须是 n_dims。  
- 变量名中不要出现 num 或 num_ 前缀；统一改为 n_xxxs 的复数形式（例如 num_layers -> n_layers）；若命名冲突，使用 n_A_xxxs 与 n_B_xxxs 区分。  
- 对于一维对象/向量，长度变量名使用 dim 表示，优先使用 xxx_dim 格式（例如 seq_dim）。  
- 对于多维张量对象，先判断其布局属于 (B,C,H,W) 或 (B,C,T,H,W) 或 (B,C,D,H,W)，并统一使用 batch, channel, time或depth, height, width 来指代对应维度；涉及具体变量时使用 xxx_batch / xxx_channel / xxx_time / xxx_height / xxx_width / xxx_depth 这类形式。
```
